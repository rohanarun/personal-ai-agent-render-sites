FAQ

The trust ledger should feel like an operating system for agent review, not an analytics dashboard.

Is a trust ledger different from an audit log?

Yes. Audit logs are usually raw event history. A trust ledger is a structured, user-facing work record with intent, evidence, result, exception, and approval state.

Who reviews the ledger?

The first reviewer is usually an operator, founder, support lead, or assistant user. Technical teams can still drill into raw logs when needed.

Does every task need screenshots?

No. Screenshots are most useful when browser state matters. Simple message or research tasks may only need source links and a concise evidence note.

What should teams build first?

Start with a stable receipt schema, approval states, source links, exception notes, and a memory gate that only saves approved outcomes.
