FAQ

The trust ledger is not extra ceremony. It is the interface that makes browser-agent work safe to approve.

Is this just logging?

No. Logs are usually for engineers. A trust ledger is a user-facing work record that links intent, evidence, action, exception, and approval.

How is it different from memory?

Memory is what the agent reuses later. The trust ledger is the proof layer that decides which memories are reliable enough to preserve.

Where does messaging fit?

A text-message AI assistant can trigger browser work, then return the receipt summary back to the user for fast review.

What should teams build first?

Start with a receipt schema, source bundle, screenshot capture, exception note, approval status, and a durable link back to the work object.
