How to set an attention budget for a text message AI agent

A text-message AI agent becomes useful when it understands attention as a scarce resource. The right budget tells it which events deserve an immediate ping, which decisions can wait for a digest, and which noisy signals should never reach you at all.

Give the agent a budget before you give it permission.

Most failed personal agents do not fail because they cannot act. They fail because every alert feels equally important. An attention budget fixes the operating agreement.

Start with interruption classes, not app categories.

Do not say that calendar, email, delivery, or finance events are always urgent. Instead define the conditions that justify interruption: time sensitivity, irreversible consequences, human waiting cost, and confidence in the source.

Set a daily cap.

Choose a maximum number of proactive pings per day. A strong default is three immediate interruptions, one midday digest, and one end-of-day review.

Budget by consequence.

Let the agent interrupt when delay creates cost: a missed ride, a customer waiting, a payment risk, a family logistics issue, or a task that blocks another person.

Require source proof.

Before an agent escalates, it should cite where the signal came from and whether it has seen conflicting evidence.

Make silence explicit.

The agent needs permission to do nothing. Add rules for marketing emails, repeated reminders, low-confidence summaries, and alerts outside your active projects.

Four modes keep a personal agent calm under pressure.

Use these modes as a horizontal policy. They work whether your agent watches SMS, email, calendar, browser work, or human-in-the-loop task updates.

Interrupt

For urgent, high-confidence items that degrade quickly if ignored.

Batch

For important items that matter today but do not need a real-time notification.

Escalate

For ambiguous items where another human or tool must verify before action.

Archive

For low-value noise that should be logged without consuming attention.

Roll it out in one week, then tighten the rules.

The first version should be simple enough to audit. If the agent cannot explain why it interrupted you, the rule is not ready.

Day one: write the interruption contract.

List the five situations where interruption is welcome. Keep the list small. Text-message agents get better when their permission surface is narrow and legible.

Day two: define quiet hours and bypasses.

Quiet hours should be strict. Bypasses should be rare and named: family emergencies, direct customer commitments, security events, or time-boxed logistics.

Day three: add confidence gates.

Require two signals for non-obvious urgency. For example, a calendar change plus a message from the attendee is stronger than either signal alone.

Day seven: score every interruption.

Mark each ping as useful, too early, too late, unnecessary, or missing. Feed those examples back into the agent prompt and operating policy.

The practical attention-budget checklist

  • Define no more than five immediate interruption cases.
  • Set a daily proactive ping cap and separate it from user-requested replies.
  • Use quiet hours with named bypass categories, not vague importance language.
  • Require the agent to include source, confidence, and recommended next step in urgent texts.
  • Batch medium-importance items into one digest instead of many separate messages.
  • Log archived items so you can find false negatives later.
  • Review missed and unnecessary alerts weekly and update the prompt with concrete failures.

Prompt clause you can adapt

Before sending a proactive text, classify the event as interrupt, batch, escalate, or archive. Interrupt only when the event is time-sensitive, confidence is high, and waiting creates real cost. Include the source, the reason for urgency, and the smallest useful action I can take.

Questions teams ask before turning the agent loose.

Attention budgets are not just a preference layer. They are the policy layer that decides whether an autonomous assistant feels trustworthy.

Should the budget be strict?

Yes at first. A strict cap creates visible tradeoffs. Once the agent has a history of useful choices, you can add context-aware exceptions.

What should happen when the cap is reached?

The agent should move non-critical items into the next digest and only bypass the cap for named emergency categories.

Can a text agent act without asking?

Only for reversible actions that you have explicitly approved. Anything irreversible should require confirmation or escalation.

How do you improve the prompt?

Use real failures. Tell the system prompt what went wrong, why the alert was premature or missing, and what should always happen next time.
