How-to guide

The weekly review turns consent memory into an operating system.

A personal AI agent's consent ledger should not be a graveyard of old corrections. The weekly review is where operators make memory durable, scoped, and safe. It is also where teams find approval fatigue, excessive blocking, and rules that no longer match the operator's real preferences.

Start with the rule that changed behavior most.

Sort ledger entries by consequence, not chronology. Changes that moved actions from ask to proceed, or from proceed to block, deserve the first pass because they alter the agent's operating freedom.

Prune broad scope

Turn global rules into scoped rules by person, project, channel, task type, or date range.

Expire temporary rules

Remove travel, launch, customer, and family-context exceptions that no longer apply.

Repair noisy lanes

If the agent asks too often, batch more. If it acts too freely, add ask or block lanes.

Workflow

A practical review sequence for operators.

Keep the ritual short, but make it concrete. The goal is to improve the next week of agent behavior, not debate every memory entry forever.

1. Review high-consequence changes.

Start with entries that affect external communication, private context, browser work, spending, or reputation.

2. Compare lane movement.

Look for rules that moved tasks between proceed, batch, ask, and block. Confirm the movement still matches your risk tolerance.

3. Narrow the widest rules.

Broad rules are where consent drift hides. Add context and expiration wherever the rule is too general.

4. Sample receipts.

Check whether the agent's receipts prove the right evidence. If receipts are vague, the rule is not reviewable.

5. Push urgent fixes to text.

If a rule needs immediate approval or reversal, send it through the operator's fastest channel rather than waiting for another dashboard session.

Review outcomes

Every ledger entry should leave the review with one clear outcome.

Keep

The rule is still useful and properly scoped.

Narrow

The rule applies, but only in a smaller context.

Expire

The temporary reason is gone.

Escalate

The rule needs immediate text approval.

FAQ

Common weekly review questions.

How long should the review take?

For a personal agent, fifteen to thirty minutes is enough if entries are already sorted by consequence and stale scope.

Should the agent recommend changes?

Yes, but recommendations should be grouped by lane movement and evidence quality so the operator can accept or reject quickly.

What should happen after review?

Accepted changes should write receipts, narrowed changes should update scope, and urgent changes should route to text approval immediately.
