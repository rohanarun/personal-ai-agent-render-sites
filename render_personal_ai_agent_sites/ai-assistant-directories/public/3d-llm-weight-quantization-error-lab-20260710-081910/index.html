Quantization trades numerical resolution for smaller, cheaper representations

A floating-point weight can take many representable values. Integer quantization maps a range of weights onto a smaller codebook, stores the integer codes, and reconstructs approximate values with a scale and sometimes a zero point.

This lab performs post-training weight-only quantization on a visible 8-by-8 teaching matrix. Symmetric mode centers codes around zero and uses the maximum absolute range. Asymmetric mode maps a minimum and maximum onto unsigned codes. Groupwise mode computes separate parameters for contiguous groups, allowing local ranges to fit the available levels more closely.

Bit depth controls codebook size. Four bits provide sixteen possible unsigned codes, while eight bits provide 256. Lower bits reduce payload but increase the distance between representable values. The idealized ratio compares only 16-bit weight payload with packed integer payload; real formats also store scales, zero points, alignment, metadata, and sometimes outlier values.

Outliers stretch a shared range. Most ordinary weights then occupy a small portion of the codebook. Grouping can isolate the outlier, and clipping can sacrifice its exact magnitude to improve resolution for the majority. Neither is automatically best: the right error objective depends on how the layer uses each weight and activation.

The scene never claims that matrix reconstruction error equals model quality. Perplexity, task accuracy, calibration, activation interactions, layer sensitivity, and hardware kernels must be evaluated on the actual model. The lab isolates the arithmetic so those larger questions start from something inspectable.

The quantization contract

Choose a representable range, map values to integer levels, store parameters needed to reconstruct them, and measure the error introduced.

Fewer levels. Explicit error.

Outliers consume resolution

A single extreme value can enlarge a tensor-wide scale, making adjacent codes too far apart for the dense center of the distribution.

Groups localize ranges

Smaller groups adapt scales to local statistics but increase scale metadata and can complicate packing and kernels.

Clipping changes the objective

A clipped outlier receives large error so common values can receive smaller error. Choose clipping against downstream quality, not appearance alone.

Hardware decides usefulness

A compact format helps only when kernels, memory layout, conversion, and device instructions exploit it efficiently.

Choose a range

Estimate minimum and maximum values per tensor, channel, or group. Calibration can use weights alone or activation observations, depending on the quantization method.

Derive scale and zero point

Scale converts code increments into real-value increments. An asymmetric zero point shifts the integer grid so a real zero remains representable.

Round into the codebook

Each scaled value is rounded and clamped to the legal integer range. Rounding creates granular error; clipping creates saturation error.

Dequantize for comparison

The lab reconstructs each integer using its group parameters. Inference kernels may multiply quantized values directly or convert blocks during computation.

Evaluate where error matters

Matrix MSE is easy to inspect but not sufficient. Practical validation measures layer outputs and end-to-end model behavior across representative tasks and contexts.

Open the design choices behind a quantized model

Per-tensor quantization

One scale covers the complete tensor. Metadata is tiny and implementation is simple, but layers with mixed ranges or outliers can lose substantial resolution.

Global

Groupwise quantization

Each block receives its own scale and zero point. Local adaptation lowers reconstruction error while adding metadata and packing constraints.

Local

Activation-aware methods

Some weights matter more because they interact with large or frequent activations. Activation-aware calibration protects important channels instead of minimizing weight error uniformly.

Signal

Quantization-aware training

Fake quantization during training exposes the model to discretization, allowing parameters to adapt before deployment. It costs training effort but can recover accuracy at low precision.

A practical quantization checklist

Confirm whether weights, activations, KV cache, or all three are quantized.
Record bit width, signedness, group axis, group size, scale type, and zero points.
Inspect outlier channels and layer sensitivity before choosing one global recipe.
Include scale, zero-point, padding, and alignment overhead in memory calculations.
Benchmark kernels on the exact hardware, batch sizes, and sequence lengths.
Evaluate perplexity and task quality, not only weight reconstruction MSE.
Test calibration data for language, domain, and long-context coverage.
Verify mixed-precision exceptions and dequantization accumulation type.
Compare throughput, latency, memory, power, and quality together.
Keep a reproducible full-precision baseline and exact conversion configuration.

Frequently asked questions

Does four-bit quantization make a model exactly four times smaller?

The raw weight payload is one quarter of FP16, but practical formats also store scales, zero points, packing alignment, unquantized layers, and runtime buffers. Report complete model and serving memory.

Why can low weight MSE still hurt model quality?

Weights do not contribute equally. Error in a sensitive channel or layer can alter downstream activations more than larger error elsewhere. End-to-end evaluation is required.

What is the difference between symmetric and asymmetric quantization?

Symmetric quantization centers a signed codebook around zero. Asymmetric quantization shifts the grid with a zero point, often fitting skewed ranges better at the cost of extra parameters and arithmetic.

Why use groupwise scales?

Groups prevent one range from controlling an entire tensor. Smaller groups usually lower local reconstruction error but add scale metadata and kernel complexity.

Is clipping always harmful?

Clipping deliberately increases error for extreme values. It can reduce total or task-weighted error by improving resolution for common values, but the clipping rule must be calibrated and evaluated.

Is this an implementation of GPTQ, AWQ, or SmoothQuant?

No. The lab teaches direct uniform quantize-dequantize arithmetic. GPTQ, AWQ, SmoothQuant, and related methods use additional optimization or activation information beyond this transparent baseline.
