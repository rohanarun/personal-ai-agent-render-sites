Vocabulary size trades sequence length for specificity.

More merges shorten frequent strings and reduce transformer positions. Rare names, code, and underrepresented languages may still fragment. Vocabulary entries also consume embedding and output-head parameters.

Token boundaries shape model cost.

Attention operates over tokens, not characters. Twice as many tokens consume twice the context budget and roughly twice the KV-cache positions. Measure with the model’s real tokenizer.

Training data leaves fingerprints.

Frequent forms earn compact representations. Underrepresented scripts and domains can split into more pieces. This compression artifact affects latency and available context even when visible text length is similar.

Questions worth asking

Is BPE the only LLM tokenizer?

No. WordPiece, Unigram Language Model, byte-level BPE, and model-specific hybrids are common.

Why begin with bytes?

A fixed byte alphabet can represent any input without unknown symbols, though one visible Unicode character may initially occupy several bytes.

Is the greedy merge globally optimal?

No. Pair frequency is BPE’s local heuristic. Corpus composition, normalization, pre-tokenization, and vocabulary size influence the result.

Primary sources

Sennrich, Haddow, and Birch (2016) applied BPE to open-vocabulary translation. Kudo (2018) introduced subword regularization and Unigram alternatives. GPT-2’s published encoder shows byte mapping and ranked merges. Hugging Face Tokenizers documentation describes production tokenization pipelines.
