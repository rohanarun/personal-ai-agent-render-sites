The router creates conditional computation.

A learned gate produces expert probabilities for each token. Top-k keeps only the strongest routes, so the model can contain many expert parameters while activating a small fraction per token.

p(expert | token) = softmax(router(token) / temperature)

Capacity makes routing a systems constraint.

Each expert receives a finite token budget. Popular experts overflow; implementations may drop assignments, reroute them, or increase capacity and communication cost.

capacity = ceil(tokens * k / experts * factor)

Top-1 is cheap and brittle.

One route minimizes expert compute and communication. A single overloaded choice can drop the token unless the system allows fallback routing.

Top-2 adds redundancy.

Two experts can combine specialties and improve gradient coverage, but they double expert activations and raise the demand on capacity.

Balance losses shape the router.

An auxiliary objective discourages collapse onto a few experts. The goal is not perfectly equal traffic at every step, but enough utilization for experts to learn distinct, useful functions.

Follow one dispatch cycle.

Score

Project token representations into router logits, then normalize them across experts.

probs = softmax(W_router x / T)

Select

Keep only the top one or two expert probabilities and renormalize their mixture weights.

routes = topk(probs, k)

Dispatch

Group token assignments by expert, respecting each expert's capacity budget.

expert_batch[e] = tokens[routes == e]

Combine

Return expert outputs to token order and weight them by the selected router probabilities.

y = sum(route_weight * expert(x))
